# 13 - 数据库迁移与数据治理

> `04-data-model/` 侧重"如何设计表 / 选数据存储"，本规范侧重"如何安全地变更和治理"。生产数据库出错代价极高，变更必须可审计、可回滚、可复现。

## 何时读哪个

| 当前任务 | 读这个 |
|---------|--------|
| 写 migration 文件、新增/修改字段、保证滚动发布兼容 | `migration.md` |
| 大表 DDL、Backfill 历史数据 | `online-ddl.md` |
| 设计数据保留策略、应对 GDPR/个保法、配置数据库账号权限 | `data-governance.md` |

## 核心原则（全局）

1. **Schema 即代码**：所有变更以 migration 文件形式提交，不允许手工改表
2. **可回滚优先**：每个 migration 必须有配套的 `down`
3. **向后兼容**：滚动发布期间新旧代码能同时工作
4. **大表无锁变更**：百万级以上的表禁用锁表 DDL
5. **数据即资产**：敏感数据分级，保留有期限

## 强制约束（不可违反）

- 生产 schema 变更必须走 migration 文件，禁止手改表
- 大表变更必须用在线 DDL 工具
- 变更必须兼容滚动发布（expand-contract）
- PII 字段加密存储（详见 `11-security/privacy-audit.md`）
- 测试环境禁止用生产数据明文
