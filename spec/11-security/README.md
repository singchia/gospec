## 核心原则（全局）

1. **默认拒绝**：权限、网络、文件访问默认关闭，按需开放
2. **最小权限**：账号、token、容器、数据库用户都遵循最小权限
3. **纵深防御**：单点失效不应导致整体沦陷
4. **不信任客户端**：所有客户端输入都视为潜在恶意
5. **审计可追溯**：敏感操作必须留痕

## 强制约束（不可违反）

- 密码必须用 `bcrypt`（cost ≥ 10）或 `argon2id`，禁止 MD5/SHA1
- SQL 全部参数化，禁止字符串拼接
- 密钥禁止进代码仓库，禁止打入镜像
- 容器以非 root 运行，镜像用 distroless 或 minimal base
- 多租户接口强制 `tenant_id` 过滤
- CI 必须包含 `govulncheck` + 依赖/镜像漏洞扫描（详见 `08-delivery/cicd.md`）
