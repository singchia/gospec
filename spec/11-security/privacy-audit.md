# 11.4 - 隐私、审计与合规

> 适用：处理个人信息（PII）、设计审计日志、应对 GDPR/个保法合规需求。

## PII 分级

| 级别 | 示例 | 处理要求 |
|------|------|---------|
| L1 公开 | 用户名、头像 | 无特殊要求 |
| L2 一般 | 邮箱、城市 | 传输加密 |
| L3 敏感 | 手机号、地址、IP | 字段加密 + 访问审计 |
| L4 高敏 | 身份证、银行卡、密码 | 加密 + 脱敏 + 最小化访问 |

## 合规要求要点

| 权利 | 实现 |
|------|------|
| **访问权** | 提供用户数据导出 API |
| **更正权** | 提供用户信息修改入口 |
| **删除权** | 硬删除或完全匿名化 |
| **可携权** | 导出标准格式（JSON/CSV） |
| **拒绝权** | 用户可关闭非必需数据处理 |

数据保留与跨境传输策略详见 `13-database-migration/data-governance.md`。

## 用户数据删除流程

```
1. 用户发起删除请求
2. 验证身份
3. 异步任务：
   - 硬删除 PII 字段或整行
   - 关联数据匿名化（保留统计价值）
   - 审计日志保留（合规需要）
4. 通知用户完成
5. 备份到期前，备份副本也会被清理
```

## 测试环境数据

- ❌ 禁止从生产直接拷贝未脱敏数据到测试环境
- ✅ 使用脱敏工具：`pg_anonymizer` / 自写脚本
- ✅ 或使用合成数据（`faker`）

```go
func anonymizeUser(u *model.User) {
    u.Email = fmt.Sprintf("user%d@example.invalid", u.ID)
    u.Phone = fmt.Sprintf("138%08d", u.ID)
    u.RealName = "Test User"
    u.IDCard = ""
}
```

## 审计日志

### 必须记录的事件

- 登录 / 登出 / 登录失败
- 权限变更（角色分配、密钥创建）
- 敏感数据访问（用户列表导出、配置修改）
- 管理员操作

### 日志格式

```json
{
  "time": "2026-04-14T10:30:45Z",
  "actor": {"id": 123, "type": "user", "ip": "1.2.3.4"},
  "action": "edge.delete",
  "resource": {"type": "edge", "id": 456},
  "result": "success",
  "trace_id": "abc..."
}
```

### 存储要求

- 独立存储（与业务日志隔离）
- 不可篡改（追加写、定期归档到 WORM 存储）
- 保留时长 ≥ 180 天（合规要求可能更长）

## 自查

- [ ] PII 字段已分级，L3+ 加密存储
- [ ] 日志输出已脱敏（见 `10-observability/logging.md`）
- [ ] 测试环境无生产明文数据
- [ ] 敏感操作有审计日志
- [ ] 提供用户数据导出 / 删除入口
